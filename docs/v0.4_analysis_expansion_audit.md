# PhageFlow v0.4 Analysis Expansion Audit

Date checked: 2026-07-01

This audit ranks candidate analyses after the validated v0.3 pipeline state. The goal is a comprehensive phage-analysis workflow with two tiers:

- `lightweight`: default or near-default analyses that run locally without large databases.
- `heavy optional`: publication-oriented modules that require extra executables, databases, or larger compute.

The default workflow should remain fast, reproducible, and validated. Heavy analyses should be opt-in, versioned, summarized conservatively, and validated through artifact contracts.

## Current Coverage

| Area | Current PhageFlow coverage | Tier |
| --- | --- | --- |
| Input validation and sample metadata | FASTA, directory, or samplesheet normalization with local reference roles | lightweight |
| Genome QC | Length, contigs, N50, GC, ambiguous bases, exact terminal-repeat heuristics | lightweight |
| Lightweight gene prediction | Dependency-light ORF/FFN/FAA/GFF outputs for smoke tests and scaffolding | lightweight |
| Codon usage | Codon counts, GC/GC3, host-linked codon/RSCU comparison when hosts are supplied | lightweight |
| Cohort redundancy | SHA256 duplicate flags and k-mer similarity | lightweight |
| Whole-genome similarity | BLASTN reciprocal coverage-aware intergenomic similarity matrices and figures | lightweight |
| Local reference context | User-supplied reference rows, nearest-reference metrics, limitations | lightweight |
| Pangenome | MMseqs default plus RBH-BLASTP sensitivity backend | lightweight-to-medium |
| Marker tree | Optional user/annotation marker proteins, MAFFT/IQ-TREE/simple engine, topology checks | optional |
| Host context | Supplied host metadata, composition, codon adaptation, optional spacer matching | optional |
| Optional annotation and screens | tRNAscan-SE, BACPHLIP, ABRicate, CheckV, Pharokka, geNomad, Phold, clinker wrappers | heavy optional |
| Optional artifact QA | Summary/package/validator coverage for optional artifacts | lightweight QA |

## External Pipeline/Tool Signals

| Source | Relevant scope | PhageFlow implication |
| --- | --- | --- |
| Pharokka | Rapid standardized phage annotation for genomes/metagenomes; docs note PHANOTATE as the default phage-tailored gene caller. | Keep Pharokka as the preferred publication annotation route; parse summary artifacts rather than replacing it. |
| Phold | Structure-informed phage annotation using protein structural homology. | Keep heavy optional; improve artifact/report summaries before adding more structure tools. |
| iPHoP | Host-genus prediction from phage genomes with a database-backed command-line workflow. | High-value heavy optional candidate; needs explicit database path, confidence summary, and limitations. |
| PhaBOX/PhaBOX2 | Identification, lifestyle prediction, taxonomy classification, host prediction, and visualization; local version is intended for larger runs. | Useful design reference, but much overlaps discovery/metagenomics. Consider only an optional wrapper or report-import path. |
| multiPhATE2 | Gene finding, functional annotation, and gene-by-gene comparisons across phage genomes. | Design reference for annotation comparison; PhageFlow should not copy it wholesale because Pharokka/MMseqs/clinker already cover much of the same space. |
| nf-core/phageannotator | Nextflow pipeline for identifying, annotating, and quantifying phage sequences in genomic/metagenomic data. | Useful for modular design and discovery-oriented gaps; PhageFlow should stay isolate/small-cohort focused unless a separate discovery mode is created. |

Sources: Pharokka GitHub and docs, Phold GitHub, iPHoP publication/Bitbucket/nf-core module, PhaBOX paper/GitHub, multiPhATE2 GitHub/publication, and nf-core/phageannotator docs.

## Recommended v0.4 Additions

| Rank | Addition | Tier | Why | Implementation boundary |
| --- | --- | --- | --- | --- |
| 1 | Complete optional-artifact summary coverage | lightweight QA | Improves both lite and heavy runs without new dependencies. | Implemented first pass: tRNAscan-SE, BACPHLIP, and ABRicate now join CheckV/Pharokka/geNomad/Phold/clinker in `optional_tool_summary.tsv`. |
| 2 | Parse high-level Pharokka/geNomad/CheckV/BACPHLIP summary fields | lightweight QA over heavy outputs | Users need report-level summaries after heavy runs, not just artifact presence. | Parse only stable summary files, counts, confidence/status fields, and limitations; avoid printing full annotation tables. |
| 3 | Add optional iPHoP wrapper | heavy optional | Host prediction is the most important missing analysis after supplied-host context. | `--run_iphop true --iphop_db PATH`; summarize top predictions/confidence conservatively; no host-range claims. |
| 4 | Add optional PhaBOX/PhaBOX2 wrapper or import mode | heavy optional | Broad taxonomy/lifestyle/host context in one tool, useful for users with databases. | Prefer import/summary path first unless CLI contract is stable in local install. |
| 5 | Add completed-run pangenome concordance into reports | lightweight QA | Existing `pangenome-sensitivity` is useful but external to reports. | Let `BUILD_REPORT` accept an optional sensitivity TSV or add a report-import utility. |
| 6 | Add protein-cluster functional category summaries from consistent annotation | heavy optional summary | Publication users need functional category counts from Pharokka/PHROG-style outputs. | Summarize categories/counts only when Pharokka outputs exist; avoid assigning biology from the lightweight ORF predictor. |

## Defer

| Candidate | Reason |
| --- | --- |
| Raw-read termini/packaging inference | Requires read inputs and a separate validation dataset. Current FASTA-only terminal-repeat heuristics are appropriate for lightweight mode. |
| Metagenomic viral-contig discovery as default | Changes the product from genome-analysis pipeline to discovery pipeline. Could become a separate mode later. |
| Automated host-range claims | Computational host predictions are evidence, not infectivity or host-range proof. |
| Copying whole external pipelines | Increases maintenance risk and duplicates mature tools. Prefer wrappers, importers, and summarized artifacts. |

## Immediate Next Step

After the optional-summary expansion, the next best implementation step is a stable optional-summary parser pass for heavy-tool outputs already wrapped by PhageFlow:

1. CheckV quality-summary counters.
2. BACPHLIP log/probability counters.
3. geNomad top-level summary/taxonomy artifact counters.
4. Pharokka feature/category artifact counters.
5. Phold confidence-table counters.

Each parser should add compact columns or a companion `optional_tool_metrics.tsv`, keep raw annotations out of the main report, add validator checks, and include regression tests with synthetic tiny artifacts.
