- MMseqs pangenome validation.
- RBH-BLASTP pangenome validation.
- Single-genome no-pangenome validation.

- Pangenome sensitivity comparison.
- Report/QA package creation.

## Current Boundaries

- Lightweight default behavior remains enabled by default.
- Heavy optional modules remain disabled by default.
- iPHoP requires an explicit `--iphop_db` path and is treated as computational host-prediction evidence only.
- No host-range, wet-lab, engineering, or manuscript-grade biological claims are added.

## Next Best Work

1. Add a release-scope checklist that decides whether future analyses are default, heavy optional, import-only, or deferred.
2. Keep PhaBOX/PhaBOX2 wrapper execution deferred until the local CLI/database contract is stable.
3. Keep vConTACT2-style wrapper execution deferred unless a stable database and validation dataset are provided.
4. Defer metagenomic discovery and read-based termini inference until separate input contracts and validation data exist.
