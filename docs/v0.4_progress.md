## Full Local Validation

After the PhaBOX/PhaBOX2 wrapper implementation, `bash bin/run_local_validation.sh` completed successfully. The run covered:

- MMseqs pangenome validation.
- RBH-BLASTP pangenome validation.
- Single-genome no-pangenome validation.
- Marker-tree validation.
- Completed-run summary utilities.
- Safety, optional-tool, and structural summaries.
- Pangenome sensitivity comparison.
- Report/QA package creation.

## Current Boundaries

- Lightweight default behavior remains enabled by default.
- Heavy optional modules remain disabled by default.
- iPHoP requires an explicit `--iphop_db` path and is treated as computational host-prediction evidence only.
- PhaBOX2 requires an explicit `--phabox_db` path and is treated as optional external-tool context only.
- No host-range, wet-lab, engineering, or manuscript-grade biological claims are added.

## Next Best Work

1. Keep vConTACT2-style wrapper execution deferred unless a stable database and validation dataset are provided.
2. Consider public-database taxonomy assignment only after database/version policy is defined.
3. Defer metagenomic discovery and read-based termini inference until separate input contracts and validation data exist.
