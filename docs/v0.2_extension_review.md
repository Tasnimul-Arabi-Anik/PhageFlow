# PhageFlow v0.2 Extension Review

This review gates proposed additions before implementation. The goal is to strengthen PhageFlow as a reproducible software pipeline while avoiding scope creep into project-specific biological interpretation.

## Recommendation

Proceed with a v0.2 software-facing extension cycle. Do not add network-dependent public reference discovery, raw-read termini analysis, or heavy host-defense/prophage modules to the core pipeline yet. Keep those as optional future modules or external project analyses.

## Worth Adding Now

| Addition | Decision | Reason |
| --- | --- | --- |
| `phageflow summarize` | Add | High value, low risk. Reuses completed output directories to produce sanitized QA summaries, marker counts, artifact counts, and validation status without rerunning workflows. |
| `phageflow package` | Add | High value, low risk. Bundles report, manifests, checksums, logs, and selected figures/tables into a shareable release archive. |
| Sanitized artifact inventory | Add | Already proven useful in final validation. Codify anonymous IDs, sizes, row/column counts, checksums, and generic PASS/FAIL/WARN/ERROR marker counts. |
| Termini/completeness heuristics from FASTA | Add carefully | Useful as assembly-QA metadata when limited to direct terminal-repeat/circularization heuristics. Do not infer wet-lab packaging mechanism from this alone. |
| Safety-screen consolidation | Add | Optional tools already exist. A consolidated report table can summarize whether AMR/virulence/integrase/toxin-like signals were detected, while leaving interpretation conservative. |
| Marker-tree provenance table | Add | Existing marker workflow tracks method notes and topology checks. Formalizing alignment engine, trimming, tree engine, bootstrap setting, and marker coverage improves validation. |
| Claim-evidence matrix | Add | Converts report outputs into a software-facing evidence map: claim label, supporting file, validation state, and limitation. |
| Container smoke test | Add | Valuable release check for a small toy dataset and output contract. |

## Add Later If Needed

| Addition | Decision | Reason |
| --- | --- | --- |
| Local reference discovery/dereplication | Defer or keep local-only | Useful, but should accept user-provided local FASTA/metadata only. Network fetching adds instability and changes the reproducibility boundary. |
| Pangenome sensitivity comparison | Add as completed-run utility | Running both methods in one workflow can double compute. Compare two completed run directories instead. |
| Structural annotation summary | Add as completed-run artifact inventory | Tool output layouts can vary, so summarize artifact classes, table shapes, sizes, checksums, and statuses without printing annotation values. |
| Synteny report integration | Defer | Clinker optional output already exists. Improve packaging first, then add static synteny export if needed. |
| Expanded host-context modules | Defer | CRISPR spacer matching and codon context already exist. Defense/prophage modules require heavier dependencies and can blur core scope. |

## Do Not Add To Core v0.2

| Addition | Decision | Reason |
| --- | --- | --- |
| Network-driven public reference discovery | Avoid in core | Makes runs non-reproducible across time, depends on external services, and mixes workflow validation with live data-mining. |
| Raw-read termini/packaging inference | Avoid until read inputs are formalized | FASTA-only heuristics are fine; read-backed termini analysis needs a separate input contract and validation dataset. |
| Biological conclusion generator | Avoid | PhageFlow should produce validated artifacts and conservative summaries, not manuscript conclusions. |
| Host-range, engineering, or wet-lab design logic | Avoid | Outside validated software-analysis scope. |

## Implementation Order

1. Add `summarize` and `package` commands.
2. Add reusable artifact-inventory/checksum code.
3. Add FASTA termini/completeness heuristic fields to QC outputs.
4. Add safety-screen consolidation from existing optional outputs.
5. Add marker-tree provenance output and validator checks.
6. Add claim-evidence matrix to `99_report`.
7. Add completed-run pangenome sensitivity comparison.
8. Add focused tests and update quickstart/optional-module docs.
9. Add container smoke-test documentation or script.

## Acceptance Criteria

- Existing v0.1 validation still passes.
- New commands can run against a completed output directory without rerunning Nextflow.
- New summaries avoid full paths, sequence strings, and table cell dumps unless explicitly requested by a local user.
- Report contract remains stable for single-genome and small-cohort runs.
- Documentation clearly separates software validation from biological interpretation.
