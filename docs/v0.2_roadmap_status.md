The container smoke test passed after two release-polish fixes: the Docker build context is reduced with `.dockerignore`, and the container runs with the host UID/GID so output files remain writable by host-side validators.

The full local validation suite passed after the v0.2 utility additions. It regenerated and validated MMseqs, RBH, single-genome, and marker-tree outputs, then exercised summarize, safety-summary, structural-summary, pangenome-sensitivity, and package utilities.

## Deferred Or Future Work

| Item | Status | Reason |
| --- | --- | --- |
| Local reference discovery/dereplication | Deferred | Should be local-input-only and needs a clear metadata/input contract before implementation. |
| Static synteny figure export | Deferred | Clinker HTML exists; static export needs tool-specific handling and visual QA. |
| Expanded host defense/prophage modules | Deferred | Heavier dependencies and project-specific interpretation risk. |
| Network-driven public reference discovery | Avoid in core | Non-reproducible across time and dependent on external services. |
| Raw-read termini/packaging inference | Avoid until reads are formalized | FASTA-only exact-repeat heuristics are implemented; read-backed inference requires a separate input contract. |

## Remaining Before Future Expansion

- Local reference discovery/dereplication remains deferred outside core v0.2 unless a local-input-only contract is defined.
- Static synteny figure export remains deferred because clinker HTML support and validator checks already exist.
- Expanded host defense/prophage modules remain deferred because they add heavy dependencies and project-specific interpretation risk.
