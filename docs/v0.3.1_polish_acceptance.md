## v0.3.1 Polish Issue

The README and quickstart say to run commands from the repository root, but many examples use paths such as `phageflow/bin/phageflow`. That works when the current directory is the parent of a clone named `phageflow`, but it fails after the common GitHub workflow:

```bash
git clone https://github.com/Tasnimul-Arabi-Anik/PhageFlow.git
cd PhageFlow
bash phageflow/bin/phageflow doctor
```

Recommended v0.3.1 polish:

- Pick one documented command context and make it explicit.
- Prefer normal clone-root commands in README/quickstart, for example `bash bin/phageflow doctor`, `nextflow run main.nf`, and paths under `assets/test_data/`.
- Keep parent-directory commands only where they are explicitly introduced as parent-directory examples.

No pipeline functionality change is needed for this polish item.

## v0.3.1 Polish Implementation

The live README, quickstart, and optional-module guide now use clone-root commands consistently:

```bash
git clone https://github.com/Tasnimul-Arabi-Anik/PhageFlow.git
cd PhageFlow
bash bin/phageflow doctor
```

The parent-directory command context above is preserved only as the historical acceptance test for `v0.3.0-validated`.
