# PhageFlow v0.4 Release Validation

Date: 2026-07-01

Release candidate: `v0.4.0-validated`

## Validation Scope

This evidence supports software/workflow validation only. It does not claim biological interpretation, host range, infectivity, wet-lab outcome, or manuscript-grade biological conclusions.

The validated default remains lightweight. Heavy tools such as CheckV, Pharokka, geNomad, Phold, clinker, iPHoP, and PhaBOX2 are disabled by default and require user-provided local executable/database resources.

## Commands

Focused checks:

```bash
python3 -m py_compile bin/*.py tests/*.py
python3 tests/test_optional_tool_summary.py
python3 tests/test_validator_optionals.py
python3 tests/test_lightweight_metrics.py
python3 tests/test_pangenome_sensitivity.py
python3 tests/test_network_context_summary.py
bash -n bin/phageflow bin/run_local_validation.sh bin/container_smoke_test.sh
git diff --check
```

Workflow validation:

```bash
bash bin/run_local_validation.sh
```

Container validation when Docker is available:

```bash
bash bin/phageflow container-smoke
```

Release metadata checks:

```bash
bash bin/phageflow doctor
bash bin/phageflow doctor --with integrated
```

## Outcomes

- Full local validation passed for MMseqs, RBH-BLASTP, single-genome no-pangenome, and marker-tree runs.
- Completed-run utilities passed for summarize, safety summary, optional-tool summaries, structural summary, pangenome sensitivity import, and package export.
- PhaBOX2 wrapper wiring passed focused validation: default runs report PhaBOX as `not_run`, and `--run_phabox true` without `--phabox_db` fails with a clear database-path error.
- Docker container smoke validation passed with Docker available.
- GitHub CI passed for baseline v0.4 wrapper commit `4efb1bd`; GitHub static CI remains the required remote gate before tagging.

## Remote Heavy-Run Readiness

Remote host `dulab` is available for future larger validations. SSH connectivity, temporary directory creation, disk availability, memory availability, and cleanup were checked non-invasively.

Observed readiness:

- Host: `dulab`
- Temporary directory policy: create under `/tmp`, copy only needed files, remove after validation.
- Approximate available resources at check time: 545 GB free disk and 95 GB available RAM.
- Remote login shell exposes Java but not Nextflow or conda/mamba in `PATH`; future heavy remote runs should bootstrap needed tools into the temporary folder or use explicit paths.

No remote heavy biological validation is claimed in v0.4 unless a database-backed run is separately executed and recorded.
